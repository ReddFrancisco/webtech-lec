Webtek Prelim Notes

The Internet

- Internet is a massive network of networks.

- It was brought online year 1969.

Nodes - The actual devices (computer,loptops).
Inter - Outside the boundaries.
Intra - Inside the boundaries
Protocols - Operate many devices.
Protocols Serial wire - One wire that sends data one at a time.
IPv4 - 32bit Address
IPv6 - 64bit Address

Gopher - Protocol desgined on retrieving documents over the internet.

- Gopher's hierarchical structure provided a platform for the first large-scale electronic library connections.

Usenet - Online bulletin boards

- Discussion system available on computers.

World Wide Web

- Invented in 1989 by Sir Tim Berners-Lee

- Retrieving documents using the Internet

- Global information medium which users can read and write via computers connected to the internet.

- The term is often mistakenly used as a synonym for the internet itself, but the web is a service that operates over the internet, as e-mail does.

WAIS(Wide Area Information Servers)

- Developed late 1980's

- WAIS is a client-server text searching system that uses ANSI standard and protocol specificationsto search index databases on remote computers.

DARPA(Defense Advance Research Projects Agency

- A military agency

- Responsible for the development of emerging technologies to be used by the military.

Circuit switch - established circuit for communication.

ARPANET(Advance Research Projects Agency Network)

Technologies that became foundation of the internet

- First network to implement the protocol suite TCP/IP.

- Early packet switching network.

IANA(Internet Assigned Numbers Authority)

- controls numbers for protocols, the Country Code Top Level Domains and maintains the IP Address allotments.

E-mail(1972)

- E-mail is a method of exchanging digital messages across people by using a device such as computer.

HTML(Hypertext Markup Language)

- Markup language used to create web pages

- focus on content and structure

- Language used to markup documents (i.e web pages) in the WWW - Structure (and content) - Presentation (mostly deprecated in favor of style sheets)
  • Web Pages
  • Webpage documents written in HTML

HTTP(Hypertext Transfer Protocol)

- Foundation of data communication for the word wide web.

- Application layer communication protocol used to access resources.

- Foundation of data communication for the world wide web.

- Uses a request-response standard protocol.

- Runs over TCP/IP, client-server architecture.

- A Stateless communications protocol.

HTTPS

- HTTP that is running on a SSL/TLS (Secure Sockets Layer/Transport Layer Security).

  • HTTP Resource Addressing

    • HTTP resources are identified using URI's(RFC3986) or more specifically, HTTP URL'S(resource locator)

    Scheme(HTTP or HTTPS) designating protocol

    Authority - uses information or authentication credentials
    DNS - Domain Name System
    Host - domain name(resolved to an IP address using DNS) of the server where the resource resides.
    Port number - path to resource (resolved relative to the document root on the server).
    - May refer to a stataic/dynamic resource
    Query - typically provided as key=value pairs, with ampersand() separators between key/value pairs.
    URN - Unified resource name
    Absolute URL - Contains protocol in the name of the website. (eg. "https://google.com")
    Relative URL - Path is generally refers to the same machine (eg. "documents/index.html")
    Fragment identifier - sometimes called as bookmark, typically use in the browsers end.

  • HTTP Request Methods

    • Standard Methods:(GET,HEAD,POST,PUT,DELETE,OPTIONS,TRACE,CONNECT)

    GET - transfer a current selected representation of the resource identified by the request URL;
    the retrieved resource is returned in the message body at the response as an entity
    HEAD - retrieves metadata about the entity implied by the request without transferring the entity itself.
    HEAD - retrieves metadata about the entity implied by the request without transferring the entity itself.
    POST - perform resource-specific processing of the entities enclosed in the message.
    PUT - store the enclosed entity in the message body under the specified request URI.
    DELETE - remove the resource associated with the request URI
    OPTIONS - request information
    TRACE - request a loop-back of the request message, used for testing/diagnostic of the request/response chain.
    TRACE - request a loop-back of the request message, used for testing/diagnostic of the request/response chain.
    CONNECT - request the establishment of a tunnel to the destination origin server.
    Content-length - usually in bytes
    Link rot - links to an external resource, there’s a possibility that somebody will rot your website and it will be a broken link.

    Safe Methods: (GET,HEAD,OPTIONS,TRACE)

    Idempotent Methods:(GET,HEAD,PUT,DELETE,OPTIONS,TRACE)
